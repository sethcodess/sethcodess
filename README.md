## 🏛️ Engineering Overview

I am a Full-Stack Software Engineer based in Nairobi, Kenya. I specialize in building robust, enterprise-grade web applications using **React**, **ASP.NET Core**, and **PostgreSQL**, deployed via **Docker** and **Azure**. 

My core focus is not just writing code, but solving business problems through clean architecture, comprehensive testing, and scalable system design. I treat software engineering as a discipline, valuing maintainability, operational excellence, and developer experience as highly as feature delivery.

## 📐 Engineering Philosophy

Professional software is read far more often than it is written. My approach is grounded in the following principles:

- **Maintainability First:** Writing code that is predictable, self-documenting, and easy for the next engineer to understand and modify.
- **Pragmatic Architecture:** Applying Clean Architecture and Domain-Driven Design (DDD) where complexity warrants it, avoiding over-engineering where it does not.
- **Testing as a Culture:** Leveraging unit, integration, and end-to-end tests to ensure reliability and enable fearless refactoring.
- **Security & Reliability:** Treating security, error handling, and system observability as fundamental requirements, not afterthoughts.
- **Business Alignment:** Understanding that technology is a tool to deliver business value; optimizing for user experience, performance, and operational cost.

## 📈 Engineering Standards & Workflow

I believe a healthy repository is the foundation of a healthy project. All my primary projects adhere to the following standards:

<details>
<summary><b>Repository Structure & Documentation</b></summary>
<br/>
Every project includes:
<ul>
  <li><code>README.md</code>: Clear setup instructions and project context.</li>
  <li><code>CHANGELOG.md</code>: Documented version history using Keep a Changelog.</li>
  <li><code>docs/adr/</code>: Architecture Decision Records capturing the <i>why</i> behind technical choices.</li>
  <li><code>docs/api/</code>: OpenAPI/Swagger specifications.</li>
</ul>
</details>

<details>
<summary><b>Development Lifecycle</b></summary>
<br/>
<ul>
  <li><b>Version Control:</b> Strict trunk-based development or Git Flow depending on project scope.</li>
  <li><b>Commits:</b> Conventional Commits (e.g., <code>feat:</code>, <code>fix:</code>, <code>refactor:</code>) for automated release notes.</li>
  <li><b>Code Review:</b> Enforced branch protections, required reviews, and automated CI checks before merge.</li>
  <li><b>CI/CD:</b> Automated build, test, and linting on every PR. Automated deployment on merge to main.</li>

</details>
